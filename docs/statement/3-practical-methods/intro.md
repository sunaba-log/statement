AIエージェントと共存しながら、主導権を握り続けるための具体的な手法です。

## 1. 仕様主導開発 (SDD 2026) の実践

- LLMとの共感覚的定義： プロンプトではなく「思想」を伝える。AIに自分の過去のプロジェクトや嗜好を学習させ、自分の「癖」を理解したエージェントを構築する手法。
- 「動く仕様書」としてのプロトタイプ： 言葉ではなく、AIに生成させたモックを触りながら、自分の直感と対話する。

## 2. マルチエージェント・オーケストレーション

- チーム・小野の編成： 「論理担当」「創造（狂気）担当」「デバッグ担当」など、役割の異なるAIエージェントを自作・配置し、自分はその「指揮官」として振る舞う技術。
- AI間の対立をデザインする： 複数のAIに議論させ、その「揺らぎ」の中から人間が最終的な解を選択するプロセス。

## 3. フィジカル・フィードバック・ループ

- Raspberry Pi / Jetson を用いた「肉体」の付与： AIをクラウドから引きずり出し、現実の物理現象（振動、光、音）と衝突させる実装パターン。
- 「ササラ棒」的インターフェースの模索： 伝統的な道具や身体技法をAIと融合させ、数値化できないユーザー体験を創出する。

## 4. ドメイン・ダイビング（領域没入）

- 製造業・現場への憑依： AIには見えない「現場の暗黙知」や「職人の勘」を、データではなく「物語」として収集し、システム設計に反映させるフィールドワーク手法。
- 越境のポートフォリオ： コードの代わりに、自分がそのドメインでいかに「あがいたか」を記録する新しいアウトプット形式。
